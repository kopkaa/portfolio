## Fullstack Developer at Sportega

Sportega (UniPortal) is a multi-tenant e-commerce platform running several sports e-shops — Sportega, Sportobchod, Fitness4u — across the CZ, SK, DE, AT, HU, PL and other markets. The architecture combines a PHP monolith with ~80 internal modules and a set of independent frontend applications (micro-frontends) integrated via Docker. I work across the stack with a focus on the frontend.

![Sportega homepage](/assets/projects/sportega/homepage.jpg)

### Key Responsibilities

**Customer Account Section** <br>
Built features in the customer section (Vue 3 + TypeScript + Vite) — the claims process including photo uploads (resolving race conditions caused by parallel Pinia store mutations), order cancellation, registration, and account management.

**Cart & Checkout** <br>
Developed the shopping cart and order flow (Vue 2 + Pinia) — cart logic, purchase form, order payments, and payment gateway integrations.

![Product detail](/assets/projects/sportega/product-detail.jpg)

**E-shop Administration** <br>
Delivered admin modules (Aurelia + TypeScript + ag-grid) for credit notes, supplier orders, menu and content management, and product exports.

**Backend Development** <br>
Implemented REST API endpoints for the customer section and domain logic in the modular core (PHP 8.3 + Nette + Doctrine) — orders, purchase gifts, catalog reindexing — along with Latte storefront templates.

**Data & Infrastructure** <br>
Worked with Elasticsearch (product catalog, synonyms), MySQL, MongoDB, Redis, and RabbitMQ. Maintained the local Docker development environment (multi-container setup with nginx, PHP-FPM, Elasticsearch, and MongoDB).

![Cart](/assets/projects/sportega/cart.jpg)

<br>

Working on a platform of this scale lets me move between modern frontend stacks and a large modular backend daily, and continuously sharpen my skills as a fullstack developer.
